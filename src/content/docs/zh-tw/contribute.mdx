我們歡迎任何規模的貢獻，無論你是初學者或專家。身為開源專案，我們相信應該要回饋貢獻者，因此很樂意對 PR、技術寫作、功能、想法提供指導。

:::tip[未解決的 Issue]
請瀏覽 [Paideia LMS 所有標記為 `help wanted` 及 `good first issues`，且未解決的 GitHub issue](https://github.com/search?q=org%3Apaideia-lms+label%3A%22good+first+issue%22%2C%22help+wanted%22%2C+state%3Aopen&type=issues)，這些程式碼儲存庫包含：Paideia LMS 程式碼、文件……有很多地方可以貢獻！
:::

想要更深入參與嗎？請參閱我們的[治理文件](https://github.com/paideia-lms/.github/blob/main/GOVERNANCE.md)，其中詳細描述了不同角色、維護者提名流程、程式碼審查流程，以及行為準則。

## 貢獻的方法

### 專案儲存庫

為 Paideia LMS 專案貢獻的方法很多！每個 Paideia LMS 儲存庫都有說明文件，連結到專案根目錄的 `CONTRIBUTING.md` 檔案。

[Paideia LMS 的 GitHub 簡介](https://github.com/paideia-lms)包含下列儲存庫：

- [Paideia LMS 主要 codebase](https://github.com/paideia-lms/Paideia)，核心學習管理系統。

- [Paideia LMS 文件](https://github.com/paideia-lms/docs)，使用 Astro Starlight 建構的文件網站！除了撰寫文件，你也可以對網站的無障礙設計、CSS、UI、UX 貢獻。文件提供多國語系，因此我們需要你的幫助翻譯整個網站。

### 貢獻的類型

除了貢獻自己的程式碼或內容，你也可以在 PR 留下審查評論、在既有的 GitHub 問題和討論中表達想法、投身我們的維護任務。這些都是巨大的貢獻！

每個 PR 都需要審查（特別是翻譯類的 PR）！想要開始貢獻、學習 Paideia LMS 相關知識嗎？審查 PR，留下評論、建議或「LGTM!」（「Looks Good To Me!」）表達贊同（譯註：「看起來很不錯！」），都是很好的方法。

我們珍視這些行為：歡迎新成員、回答問題、分享自己用 Paideia LMS 或為 Paideia LMS 寫的東西。除了傳統的 GitHub 貢獻，Paideia LMS 也認同並支持成員參與社群、在部落格文章／影片／演講分享 Paideia LMS、幫助維持健康的社群。

## 為文件貢獻

我們提供了幾份指南協助你對 Paideia LMS 文件做出貢獻。

無論你是第一次參與開源專案貢獻，或需要為你實作的新功能撰寫說明文件，亦或者是在尋找下一頁要翻譯什麼的資深譯者，再或者是想了解如何審查 PR ⋯⋯，我們都能提供協助！

請查看我們的[貢獻指南](/zh-tw/contribute/)以了解如何以各種身份對 Paideia LMS 文件做出貢獻，例如校稿人員、作家、譯者、功能開發人員，甚至是 PR 審查者。
