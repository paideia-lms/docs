## 獲取所需的先決條件

### 作業系統

Paideia LMS 運行在 **macOS**（ARM64 或 x64）或 **Linux**（ARM64 或 x64）上。確保你可以存取相容的作業系統。

### 終端

你將使用**命令列（終端）**來下載、安裝和運行 Paideia LMS，並配置環境。

你可以透過作業系統的本地終端程式存取命令列。常見的應用程式包括 **Terminal**（macOS/Linux）、**命令提示字元**（Windows）和 **Termux**（Android）。其中一些可能已經在你的裝置上了。

### PostgreSQL 資料庫

Paideia LMS 需要 **PostgreSQL** 資料庫。你需要以下之一：
- 本地 PostgreSQL 安裝
- 存取遠端 PostgreSQL 資料庫
- Docker Compose（包含 PostgreSQL）

要檢查是否在本地安裝了 PostgreSQL，請運行：

```sh
psql --version

# 輸出範例
psql (PostgreSQL) 18.0
```

如果你沒有安裝 PostgreSQL，你可以：
- 在本地安裝 PostgreSQL
- 使用 Docker Compose（我們將在安裝步驟中介紹）
- 使用雲端 PostgreSQL 服務

### S3 相容儲存

Paideia LMS 需要用於檔案儲存的 **S3 相容儲存**。你需要以下之一：
- MinIO（S3 相容儲存）- 包含在 Docker Compose 中
- S3 相容服務（AWS S3、DigitalOcean Spaces 等）
- 本地 MinIO 安裝

### 程式碼編輯器（可選）

雖然不是必需的，但**程式碼編輯器**對於編輯配置檔案很有幫助。

:::tip[我們將使用……]
本教程將使用 **VS Code**，但你可以使用適合你作業系統的任何編輯器。
:::
