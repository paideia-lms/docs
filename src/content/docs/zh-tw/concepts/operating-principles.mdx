**Paideia LMS** 在一套核心原則下營運，這些原則指導我們如何開發、維護和溝通項目。這些原則確保透明度、社區參與和長期可持續性。

## 開源且永久免費

**Paideia 在 GNU Affero 通用公共許可證 v3.0 (AGPL-3.0) 下開源且永久免費。** 此許可證確保 Paideia 對教育機構保持免費存取，同時維護開源軟體的原則。

這意味著：

✅ **免費和開源：** 自由使用、修改和分發

✅ **學校和教育機構：** 可以無限制地託管和使用

✅ **修改必須共享：** 如果你修改並託管此軟體，你必須共享你的更改

✅ **需要相同許可證：** 所有分叉和衍生品必須使用 AGPL-3.0

⚠️ **商業託管限制：** 雖然許可證允許商業使用，但我們強烈不鼓勵組織將此軟體作為付費服務為學校託管。該項目的宗旨是保持對教育機構的免費存取。

如果你對符合我們使命的商業合作夥伴關係或支援服務感興趣，請聯繫我們討論合作機會。

## 公開變更日誌

**Paideia LMS 的所有更改都在 GitHub 和文件中公開記錄。** 我們維護一個全面的變更日誌，追蹤對平台所做的每個功能添加、錯誤修復和改進。這確保了對正在更改的內容和時間的完全透明。

變更日誌在多個地方可用：
- **GitHub 發布：** 每個發布都包含詳細的發布說明
- **文件：** 變更日誌整合到文件網站中
- **GitHub 提交：** 在提交歷史中追蹤單個更改

這種透明度幫助使用者了解新內容、已修復的內容以及可能影響其安裝的內容。它還幫助社區追蹤項目進度並計劃更新。

## 季度白皮書

**每三個月，我們在文件網站上發布一份白皮書，涵蓋進度、願景和方向。** 這些白皮書全面展示了 Paideia 的現狀、過去和未來。

每份白皮書包括：

### 進度回顧

過去季度完成的工作的詳細摘要：
- 新功能和能力
- 效能改進
- 錯誤修復和穩定性增強
- 社區貢獻和參與
- 技術里程碑和成就

### 長期願景

我們對 Paideia LMS 的戰略方向和長期目標：
- 我們希望在接下來的 2-5 年內將平台帶向何方
- 我們正在規劃的主要功能和控制
- 架構改進和可擴展性目標
- 社區增長和參與目標

### 短期願景

我們的即時計劃和近期目標：
- 我們在下一季度的工作內容
- 即將推出的功能和改進
- 開發的優先領域
- 即時目標和里程碑

這些白皮書確保社區始終了解 Paideia 的方向，並可以相應地計劃。它們還為反饋和討論項目的未來提供了機會。

## 問題和討論驅動的開發

**開發由問題和討論驅動，核心團隊做出決策。** 我們相信透明、社區參與的開發，在實施之前可以公開討論想法。

### 工作原理

1. **問題：** 任何人都可以在 GitHub 上打開問題以報告錯誤、建議功能或請求改進
2. **討論：** 複雜主題在 GitHub 討論中討論，社區可以分享想法並提供反饋
3. **決策制定：** 核心團隊審查問題和討論，做出技術決策，並確定工作優先級
4. **實施：** 一旦做出決策，工作就會被實施和記錄

### 社區參與

我們積極鼓勵社區參與：
- **錯誤報告：** 幫助我們識別和修復問題
- **功能請求：** 分享你的改進想法
- **討論：** 參與技術討論和規劃
- **貢獻：** 為錯誤修復和功能提交拉取請求

### 核心團隊決策

雖然社區意見很有價值，但核心團隊對以下內容做出最終決定：
- 技術架構和設計選擇
- 功能優先級和路線圖
- 安全和穩定性問題
- 項目方向和戰略

這種平衡確保 Paideia 在保持對社區需求響應的同時，朝著明確的方向前進。

## 透明度和開放性

所有這些營運原則共同確保 Paideia LMS 的開發透明、社區驅動且可持續。我們相信，當開源項目對其流程、決策和方向保持開放時，它們會蓬勃發展。

無論你是使用者、貢獻者，還是只是對項目感到好奇，你總是可以了解正在發生的事情、計劃的內容以及如何做出決策。這種透明度建立信任並鼓勵社區參與。
