註冊是將用戶連接到 Paideia LMS 中課程的機制。每個註冊記錄建立了用戶與特定課程的關係，並定義了他們在該課程中的權限。

## 什麼是註冊？

當用戶註冊課程時，他們將獲得該課程內容和活動的訪問權限。註冊記錄存儲有關此關係的重要信息，包括：

- **用戶** - 註冊的人員
- **課程** - 他們註冊的課程
- **角色** - 用戶在課程中可以做什麼（見下面的課程角色）
- **狀態** - 註冊是活躍、非活躍、已完成還是已退學
- **註冊日期** - 用戶註冊的日期
- **完成日期** - 用戶完成課程的日期（如適用）
- **小組** - 用戶在課程內所屬的任何小組或隊列

系統確保每個用戶在任何給定課程中只能註冊一次，防止重複註冊。

## 課程角色

Paideia LMS 定義了**四種不同的註冊角色**，這些角色決定了用戶在特定課程中可以做什麼：

| 角色 | 描述 |
|------|-------------|
| **學生** | 課程參與者的標準學習者角色 |
| **教師** | 具有完整課程控制權的主要講師 |
| **教學助理（TA）** | 具有有限講師權限的支持角色 |
| **管理員** | 管理支持角色 |

### 學生

課程參與者的標準學習者角色。學生可以：

- 查看課程內容
- 提交作業
- 參加測驗
- 參與討論

### 教師

具有完整課程控制權的主要講師。教師可以：

- 管理所有課程內容
- 批改學生作業
- 管理註冊
- 訪問課程設置

### 教學助理（TA）

具有有限講師權限的支持角色。教學助理可以：

- 批改學生作業
- 查看課程設置
- 協助課程管理任務

### 管理員

管理支持角色。管理員可以：

- 管理註冊
- 訪問課程設置
- 查看報告和分析

## 註冊狀態

註冊可以根據用戶在課程中的當前狀態具有不同的狀態：

- **活躍**（默認）- 用戶正在積極註冊並參與
- **非活躍** - 註冊暫時暫停
- **已完成** - 用戶已成功完成課程
- **已退學** - 用戶已退出課程

## 註冊管理

具有適當權限的用戶可以創建和管理註冊。通常，課程教師和管理員可以：

- 向課程添加新用戶
- 更改課程中的用戶角色
- 更新註冊狀態
- 從課程中刪除用戶
- 將用戶分配到小組或隊列

## 小組成員身份

註冊可以與課程內的小組關聯，從而實現諸如部分、團隊或隊列等組織結構。這樣可以更好地組織課程和開展基於小組的活動。

## 註冊界面

註冊管理界面提供所有課程參與者的清晰視圖，顯示：

- 用戶詳細信息（姓名、電子郵件）
- 當前角色和狀態
- 最後訪問信息
- 編輯或管理註冊的選項

管理員和教師可以輕鬆添加新註冊、修改現有註冊，並通過此界面管理課程參與者。

:::note
註冊系統與系統範圍的用戶角色不同。用戶可能具有 `instructor` 的系統角色（允許他們創建課程），同時在特定課程中註冊為 `student`。相反，具有 `student` 系統角色的用戶可以在他們教授的課程中註冊為 `teacher`。

**註冊角色始終優先於課程特定權限**，這意味著在課程內，註冊角色決定用戶可以做什麼，無論其系統範圍的角色如何。
:::
