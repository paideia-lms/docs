注册是将用户连接到 Paideia LMS 中课程的机制。每个注册记录建立了用户与特定课程的关系，并定义了他们在该课程中的权限。

## 什么是注册？

当用户注册课程时，他们将获得该课程内容和活动的访问权限。注册记录存储有关此关系的重要信息，包括：

- **用户** - 注册的人员
- **课程** - 他们注册的课程
- **角色** - 用户在课程中可以做什么（见下面的课程角色）
- **状态** - 注册是活跃、非活跃、已完成还是已退学
- **注册日期** - 用户注册的日期
- **完成日期** - 用户完成课程的日期（如适用）
- **小组** - 用户在课程内所属的任何小组或队列

系统确保每个用户在任何给定课程中只能注册一次，防止重复注册。

## 课程角色

Paideia LMS 定义了**四种不同的注册角色**，这些角色决定了用户在特定课程中可以做什么：

| 角色 | 描述 |
|------|-------------|
| **学生** | 课程参与者的标准学习者角色 |
| **教师** | 具有完整课程控制权的主要讲师 |
| **教学助理（TA）** | 具有有限讲师权限的支持角色 |
| **管理员** | 管理支持角色 |

### 学生

课程参与者的标准学习者角色。学生可以：

- 查看课程内容
- 提交作业
- 参加测验
- 参与讨论

### 教师

具有完整课程控制权的主要讲师。教师可以：

- 管理所有课程内容
- 批改学生作业
- 管理注册
- 访问课程设置

### 教学助理（TA）

具有有限讲师权限的支持角色。教学助理可以：

- 批改学生作业
- 查看课程设置
- 协助课程管理任务

### 管理员

管理支持角色。管理员可以：

- 管理注册
- 访问课程设置
- 查看报告和分析

## 注册状态

注册可以根据用户在课程中的当前状态具有不同的状态：

- **活跃**（默认）- 用户正在积极注册并参与
- **非活跃** - 注册暂时暂停
- **已完成** - 用户已成功完成课程
- **已退学** - 用户已退出课程

## 注册管理

具有适当权限的用户可以创建和管理注册。通常，课程教师和管理员可以：

- 向课程添加新用户
- 更改课程中的用户角色
- 更新注册状态
- 从课程中删除用户
- 将用户分配到小组或队列

## 小组成员身份

注册可以与课程内的小组关联，从而实现诸如部分、团队或队列等组织结构。这样可以更好地组织课程和开展基于小组的活动。

## 注册界面

注册管理界面提供所有课程参与者的清晰视图，显示：

- 用户详细信息（姓名、电子邮件）
- 当前角色和状态
- 最后访问信息
- 编辑或管理注册的选项

管理员和教师可以轻松添加新注册、修改现有注册，并通过此界面管理课程参与者。

:::note
注册系统与系统范围的用户角色不同。用户可能具有 `instructor` 的系统角色（允许他们创建课程），同时在特定课程中注册为 `student`。相反，具有 `student` 系统角色的用户可以在他们教授的课程中注册为 `teacher`。

**注册角色始终优先于课程特定权限**，这意味着在课程内，注册角色决定用户可以做什么，无论其系统范围的角色如何。
:::
