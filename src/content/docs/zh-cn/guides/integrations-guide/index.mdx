Paideia LMS 支持与外部系统和服务的集成，以增强功能并简化教育机构的工作流程。

## 概览

集成允许 Paideia LMS 连接到外部工具和服务，实现无缝的数据交换、身份验证和增强功能。这对于使用多个系统并需要它们协同工作的教育机构尤其重要。

## 学习工具互操作性 (LTI)

Paideia LMS 支持**学习工具互操作性 (LTI)**，这是由 1EdTech（原 IMS Global Learning Consortium）开发的标准，可实现学习系统与外部教育工具之间的无缝集成。

LTI 允许 Paideia LMS ：

- 托管由外部或第三方系统提供的课程内容和工具
- 允许学习者访问外部工具而无需单独登录
- 与外部系统共享学习者信息和学习上下文
- 使外部工具能够将成绩结果发送回 Paideia LMS

LTI 使用 OAuth2、OpenID Connect 和 JSON Web Tokens (JWT) 进行安全的身份验证和数据交换。该标准被主要的教育内容提供商和学习管理系统广泛采用。

:::note[LTI 标准]
LTI 是一个行业标准，可实现不同教育技术平台之间的互操作性。有关更多信息，请参阅 [学习工具互操作性维基百科文章](https://en.wikipedia.org/wiki/Learning_Tools_Interoperability)。
:::

## Microsoft 集成

Paideia LMS 包含内置的 **Microsoft 集成**，支持：

### Microsoft Teams

与 Microsoft Teams 的集成允许您：

- 与 Teams 同步课程内容和活动
- 实现 Teams 和 Paideia LMS 之间的无缝协作
- 直接从 Teams 访问课程材料

### Microsoft 多因素身份验证 (MFA)

许多学校使用 Microsoft MFA 来增强安全性。Paideia LMS 支持 Microsoft MFA 配置，允许机构：

- 使用 Microsoft 的身份验证系统进行用户登录
- 利用现有的 Microsoft Active Directory (Azure AD) 基础设施
- 在系统间保持一致的身份验证策略
- 为用户提供单点登录体验

对于已经投资于 Microsoft 生态系统并希望利用其现有身份验证基础设施的教育机构来说，此集成特别有价值。

- 图书馆管理系统
- 评估和监考工具

- 分享您的用例和需求

我们致力于构建为使用 Paideia LMS 的教育机构带来真正价值的集成。您的反馈有助于我们确定优先开发哪些集成。
