**Paideia LMS** 在一套核心原则下运营，这些原则指导我们如何开发、维护和沟通项目。这些原则确保透明度、社区参与和长期可持续性。

## 开源且永久免费

**Paideia 在 GNU Affero 通用公共许可证 v3.0 (AGPL-3.0) 下开源且永久免费。** 此许可证确保 Paideia 对教育机构保持免费访问，同时维护开源软件的原则。

这意味着：

✅ **免费和开源：** 自由使用、修改和分发

✅ **学校和教育机构：** 可以无限制地托管和使用

✅ **修改必须共享：** 如果你修改并托管此软件，你必须共享你的更改

✅ **需要相同许可证：** 所有分叉和衍生品必须使用 AGPL-3.0

⚠️ **商业托管限制：** 虽然许可证允许商业使用，但我们强烈不鼓励组织将此软件作为付费服务为学校托管。该项目的宗旨是保持对教育机构的免费访问。

如果你对符合我们使命的商业合作伙伴关系或支持服务感兴趣，请联系我们讨论合作机会。

## 公开变更日志

**Paideia LMS 的所有更改都在 GitHub 和文档中公开记录。** 我们维护一个全面的变更日志，跟踪对平台所做的每个功能添加、错误修复和改进。这确保了对正在更改的内容和时间的完全透明。

变更日志在多个地方可用：
- **GitHub 发布：** 每个发布都包含详细的发布说明
- **文档：** 变更日志集成到文档网站中
- **GitHub 提交：** 在提交历史中跟踪单个更改

这种透明度帮助用户了解新内容、已修复的内容以及可能影响其安装的内容。它还帮助社区跟踪项目进度并计划更新。

## 季度白皮书

**每三个月，我们在文档网站上发布一份白皮书，涵盖进度、愿景和方向。** 这些白皮书全面展示了 Paideia 的现状、过去和未来。

每份白皮书包括：

### 进度回顾

过去季度完成的工作的详细摘要：
- 新功能和能力
- 性能改进
- 错误修复和稳定性增强
- 社区贡献和参与
- 技术里程碑和成就

### 长期愿景

我们对 Paideia LMS 的战略方向和长期目标：
- 我们希望在接下来的 2-5 年内将平台带向何方
- 我们正在规划的主要功能和控制
- 架构改进和可扩展性目标
- 社区增长和参与目标

### 短期愿景

我们的即时计划和近期目标：
- 我们在下一季度的工作内容
- 即将推出的功能和改进
- 开发的优先领域
- 即时目标和里程碑

这些白皮书确保社区始终了解 Paideia 的方向，并可以相应地计划。它们还为反馈和讨论项目的未来提供了机会。

## 问题和讨论驱动的开发

**开发由问题和讨论驱动，核心团队做出决策。** 我们相信透明、社区参与的开发，在实施之前可以公开讨论想法。

### 工作原理

1. **问题：** 任何人都可以在 GitHub 上打开问题以报告错误、建议功能或请求改进
2. **讨论：** 复杂主题在 GitHub 讨论中讨论，社区可以分享想法并提供反馈
3. **决策制定：** 核心团队审查问题和讨论，做出技术决策，并确定工作优先级
4. **实施：** 一旦做出决策，工作就会被实施和记录

### 社区参与

我们积极鼓励社区参与：
- **错误报告：** 帮助我们识别和修复问题
- **功能请求：** 分享你的改进想法
- **讨论：** 参与技术讨论和规划
- **贡献：** 为错误修复和功能提交拉取请求

### 核心团队决策

虽然社区意见很有价值，但核心团队对以下内容做出最终决定：
- 技术架构和设计选择
- 功能优先级和路线图
- 安全和稳定性问题
- 项目方向和战略

这种平衡确保 Paideia 在保持对社区需求响应的同时，朝着明确的方向前进。

## 透明度和开放性

所有这些运营原则共同确保 Paideia LMS 的开发透明、社区驱动且可持续。我们相信，当开源项目对其流程、决策和方向保持开放时，它们会蓬勃发展。

无论你是用户、贡献者，还是只是对项目感到好奇，你总是可以了解正在发生的事情、计划的内容以及如何做出决策。这种透明度建立信任并鼓励社区参与。
