## Paideia LMSを起動する

環境が設定されたので、Paideia LMSサーバーを起動できます。

<Steps>
1. Paideia LMSサーバーを起動：

   ```bash
   ./paideia server
   ```

   または単に：

   ```bash
   ./paideia
   ```

   サーバーが起動し、設定されたポート（デフォルト：バックエンドAPI用`3001`、フロントエンド用`3000`）でリッスンします。

2. サーバーが起動するまで待ちます。ターミナルでPaideia LMSが実行中であることを確認できるはずです。

   :::tip[サーバー起動]
   保留中のマイグレーションがある場合、サーバーは起動時に自動的にデータベースマイグレーションを実行します。
   :::
</Steps>

## Paideia LMSにアクセスする

<Steps>
3. ブラウザを開き、`http://localhost:3000`（または`FRONTEND_PORT`環境変数で設定されたポート）に移動します。

   Paideia LMSのログインページが表示されるはずです。

4. 初回アクセス時に、管理者アカウントの作成を求められます。画面上の指示に従って管理者ユーザーを作成してください。

   :::caution[初回セットアップ]
   管理者認証情報を忘れないでください！ログインしてPaideia LMSインスタンスを管理するために必要です。
   :::

5. 管理者アカウントを作成したら、ログインしてPaideia LMSダッシュボードにアクセスします。
</Steps>

おめでとうございます！Paideia LMSが実行中でアクセス可能になりました！

## インストールの確認

<Steps>
6. Paideia LMSインターフェースを探索：

   - ダッシュボードを確認
   - さまざまなセクションをナビゲート
   - すべての機能にアクセスできることを確認

7. APIエンドポイントを確認：

   `http://localhost:3001`に移動して、APIが実行中であることを確認します（APIドキュメントまたはヘルスチェック応答が表示されるはずです）。
</Steps>

:::tip[Paideia LMSの実行]

Paideia LMSサーバーが実行中の場合：
- `http://localhost:3000`でウェブインターフェースにアクセス
- `http://localhost:3001`でAPIにアクセス
- ターミナルで<kbd>Ctrl + C</kbd>を押すことで、いつでもサーバーを停止

サーバーは、停止するかターミナルを閉じるまで実行を続けます。
:::
