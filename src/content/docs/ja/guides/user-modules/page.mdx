**ページ**モジュールは、講師がリッチテキスト HTML を使用して静的な教育コンテンツを作成できる読み取り専用のアクティビティモジュールです。このモジュールタイプは、コース資料、講義ノート、その他の情報コンテンツを提示するのに最適です。

## 概要

ページモジュールは、ユーザーの対話や提出を必要とせずにコンテンツを提示する情報提供型（読み取り専用）モジュールです。以下に最適です：

- 講義ノートとコース資料
- 読書課題と補足コンテンツ
- コースの告知と重要な情報
- 参考資料とドキュメント

## ページモジュールの作成

ページモジュールを作成するには：

1. コースに移動します
2. 「モジュールを追加」または「アクティビティを作成」をクリックします
3. モジュールタイプから「ページ」を選択します
4. ページのタイトルと説明を入力します
5. リッチテキストエディターを使用してコンテンツを作成します
6. 準備が整ったら、下書きとして保存するか公開します

## リッチテキストエディター

ページモジュールには、以下をサポートする機能豊富なリッチテキストエディターが含まれています：

- **テキストフォーマット**：太字、斜体、下線、取り消し線
- **見出し**：複数の見出しレベル（H1-H6）
- **リスト**：順序付きリストと順序なしリスト
- **リンク**：内部リンクと外部リンク
- **画像**：代替テキスト付きのインライン画像
- **テーブル**：テーブルの作成とフォーマット
- **コードブロック**：構文ハイライト付きのコードスニペット
- **HTML**：上級ユーザー向けの直接 HTML 編集

## モジュールステータス

ページモジュールは標準のモジュールライフサイクルに従います：

- **下書き**：コンテンツが作成および編集中；作成者と許可されたユーザーのみに表示
- **公開済み**：コンテンツが学生の準備が整いました；登録学生に表示
- **アーカイブ済み**：コンテンツが保存されていますが、もはやアクティブではありません；学生には非表示

## ベストプラクティス

ページモジュールを作成する際：

- **明確な見出しを使用**して、階層的にコンテンツを整理します
- **長いコンテンツを分割**して、読みやすさを向上させます
- **画像と図表を含める**ことで理解を深めます
- **内部リンクを追加**して、コース内の関連コンテンツを接続します
- **リストを使用**して情報を明確に提示します
- **公開前にプレビュー**して、フォーマットが正しく表示されることを確認します

## 公開済みページの編集

ページモジュールが公開された後でも、編集できます。変更は保存後すぐに学生に表示されます。考慮事項：

- 透明性のために最近の更新についてのメモを追加する
- 重要なページの変更を追跡する
- 利用可能な場合は、バージョン履歴を使用して変更を確認する
