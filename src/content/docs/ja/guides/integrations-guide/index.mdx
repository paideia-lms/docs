Paideia LMS は、外部システムやサービスとの統合をサポートし、教育機関の機能を強化し、ワークフローを効率化します。

## 概要

統合により、Paideia LMS は外部ツールやサービスに接続し、シームレスなデータ交換、認証、機能強化を実現できます。これは、複数のシステムを使用し、それらを連携させる必要がある教育機関にとって特に重要です。

## 学習ツール相互運用性 (LTI)

Paideia LMS は、**学習ツール相互運用性 (LTI)** をサポートしています。これは、1EdTech（旧 IMS Global Learning Consortium）が開発した標準で、学習システムと外部教育ツール間のシームレスな統合を可能にします。

LTI により、Paideia LMS は以下が可能になります：

- 外部またはサードパーティシステムが提供するコースコンテンツやツールをホスト
- 学習者が別途ログインすることなく外部ツールにアクセス
- 学習者情報と学習コンテキストを外部システムと共有
- 外部ツールが成績結果を Paideia LMS に送信

LTI は、OAuth2、OpenID Connect、JSON Web Tokens (JWT) を使用して、安全な認証とデータ交換を行います。この標準は、主要な教育コンテンツプロバイダーや学習管理システムで広く採用されています。

:::note[LTI 標準]
LTI は、異なる教育技術プラットフォーム間の相互運用性を可能にする業界標準です。詳細については、[学習ツール相互運用性の Wikipedia 記事](https://en.wikipedia.org/wiki/Learning_Tools_Interoperability) を参照してください。
:::

## Microsoft 統合

Paideia LMS には、以下の機能をサポートする組み込みの **Microsoft 統合** が含まれています：

### Microsoft Teams

Microsoft Teams との統合により、以下が可能になります：

- コースコンテンツとアクティビティを Teams と同期
- Teams と Paideia LMS の間のシームレスなコラボレーションを実現
- Teams から直接コース資料にアクセス

### Microsoft 多要素認証 (MFA)

多くの学校がセキュリティ強化のために Microsoft MFA を使用しています。Paideia LMS は Microsoft MFA 構成をサポートし、機関が以下を可能にします：

- ユーザーログインに Microsoft の認証システムを使用
- 既存の Microsoft Active Directory (Azure AD) インフラストラクチャを活用
- システム間で一貫した認証ポリシーを維持
- ユーザーにシングルサインオン体験を提供

この統合は、既に Microsoft のエコシステムに投資しており、既存の認証インフラストラクチャを活用したい教育機関にとって特に価値があります。

## 今後の統合

より洗練されたシステムとの統合を追加し、Paideia LMS の機能を強化することを検討しています。検討中の潜在的な統合には以下が含まれます：

- 追加の認証プロバイダー
- ビデオ会議プラットフォーム
- 学生情報システム (SIS)
- 図書館管理システム
- 評価および監視ツール
- 予約およびスケジュールシステム

## 新しい統合の提案

新しい統合の提案を歓迎します！あなたの機関やより広範な Paideia LMS コミュニティに利益をもたらす統合のアイデアがある場合は、以下を自由に行ってください：

- [GitHub Discussions](https://github.com/paideia-lms/paideia/discussions) で議論を開始
- 使用例と要件を共有
- 統合がユーザーにどのように利益をもたらすかを説明

私たちは、Paideia LMS を使用する教育機関に真の価値を追加する統合を構築することに取り組んでいます。あなたのフィードバックは、次に開発する統合の優先順位を決定するのに役立ちます。
