## Start Paideia LMS

Now that your environment is configured, you can start the Paideia LMS server.

<Steps>
1. Start the Paideia LMS server:

   ```bash
   ./paideia server
   ```

   Or simply:

   ```bash
   ./paideia
   ```

   The server will start and listen on the configured ports (default: `3001` for backend API, `3000` for frontend).

2. Wait for the server to start. You should see confirmation in the terminal that Paideia LMS is running.

   :::tip[Server startup]
   The server will automatically run database migrations during startup if there are any pending migrations.
   :::
</Steps>

## Access Paideia LMS

<Steps>
3. Open your browser and navigate to `http://localhost:3000` (or the port configured via `FRONTEND_PORT` environment variable).

   You should see the Paideia LMS login page.

4. On first access, you'll be prompted to create an admin account. Follow the on-screen instructions to create your admin user.

   :::caution[First-time setup]
   Make sure to remember your admin credentials! You'll need them to log in and manage your Paideia LMS instance.
   :::

5. Once you've created your admin account, log in to access the Paideia LMS dashboard.
</Steps>

Congratulations! You now have Paideia LMS running and accessible!

## Verify your installation

<Steps>
6. Explore the Paideia LMS interface:

   - Check the dashboard
   - Navigate through the different sections
   - Verify that all features are accessible

7. Check the API endpoint:

   Navigate to `http://localhost:3001` to verify the API is running (you should see API documentation or a health check response).
</Steps>

:::tip[Running Paideia LMS]

While the Paideia LMS server is running, you can:
- Access the web interface at `http://localhost:3000`
- Access the API at `http://localhost:3001`
- Stop the server at any time by pressing <kbd>Ctrl + C</kbd> in the terminal

The server will continue running until you stop it or close the terminal.
:::
