The **Quiz** module is an interactive activity module that allows instructors to create assessments with auto-grading capabilities. This module type is ideal for tests, exams, quizzes, and other assessments that need automated scoring.

## Overview

Quiz modules are interactive modules that require user participation and track submissions. They support:

- **Multiple question types**: Multiple choice, true/false, short answer, essay, and more
- **Auto-grading**: Automatic scoring for objective questions
- **Question banks**: Create reusable question libraries
- **Randomization**: Randomize questions and answer choices
- **Timed assessments**: Set time limits for quizzes
- **Attempt tracking**: Monitor student attempts and performance

## Creating a Quiz Module

To create a quiz module:

1. Navigate to your course
2. Click "Add Module" or "Create Activity"
3. Select "Quiz" from the module types
4. Enter a title and description
5. Configure quiz settings (timing, attempts, grading)
6. Add questions to your quiz
7. Save as draft or publish when ready

## Question Types

Quiz modules support various question types:

### Objective Questions (Auto-graded)

- **Multiple Choice**: Select one correct answer from multiple options
- **Multiple Select**: Select multiple correct answers
- **True/False**: Choose true or false
- **Matching**: Match items from two columns
- **Ordering**: Arrange items in the correct order
- **Fill in the Blank**: Type the missing word or phrase
- **Short Answer**: Brief text responses (can be auto-graded with keywords)

### Subjective Questions (Manual grading)

- **Essay**: Long-form written responses
- **File Upload**: Submit files as answers
- **Audio/Video**: Record audio or video responses

## Quiz Settings

When creating a quiz, you can configure:

### Timing

- **Time limit**: Set a maximum time for completing the quiz
- **Time warnings**: Notify students when time is running out
- **Auto-submit**: Automatically submit when time expires
- **Open/Close dates**: Control when the quiz is available

### Attempts

- **Unlimited attempts**: Students can take the quiz multiple times
- **Limited attempts**: Set a maximum number of attempts
- **Single attempt**: Students can only take the quiz once
- **Attempt scoring**: Keep highest, latest, or average score

### Grading

- **Points per question**: Assign point values to questions
- **Total points**: Set the total possible points
- **Passing grade**: Define minimum passing score
- **Grade visibility**: Control when grades are visible to students
- **Feedback**: Provide immediate or delayed feedback

### Display Options

- **Question order**: Randomize question order
- **Answer order**: Randomize answer choices
- **One question per page**: Show questions one at a time
- **Navigation**: Allow or restrict navigation between questions
- **Review**: Allow students to review their answers

## Question Banks

Create reusable question banks:

- **Organize questions**: Group questions by topic or difficulty
- **Random selection**: Randomly select questions from banks
- **Share questions**: Share question banks across courses
- **Import/Export**: Import questions from files or export for backup

## Auto-Grading

Quiz modules automatically grade:

- **Objective questions**: Multiple choice, true/false, matching, etc.
- **Keyword matching**: Short answer questions with keyword matching
- **Immediate feedback**: Show results immediately after submission
- **Partial credit**: Award partial credit for partially correct answers

## Module Status

Quiz modules follow the standard module lifecycle:

- **Draft**: Quiz is being created; visible only to creator and granted users
- **Published**: Quiz is ready for students; visible to enrolled students
- **Archived**: Quiz is preserved but no longer active; hidden from students

## Best Practices

When creating quiz modules:

- **Write clear questions** that are unambiguous
- **Use appropriate question types** for the content being assessed
- **Set realistic time limits** that allow students to complete all questions
- **Provide feedback** to help students learn from mistakes
- **Review auto-graded questions** to ensure accuracy
- **Use question banks** for reusable content
- **Test your quiz** before publishing to students
